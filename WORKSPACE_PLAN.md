## Package Catalog

### 1. `packages/keyring_core` вҖ” Core API (Pure Dart)

**Purpose:** Define the platform-independent Dart API that mirrors Rust's `keyring-core`.

**Files:**
```
keyring_core/
в”ңв”Җв”Җ pubspec.yaml
в””в”Җв”Җ lib/
    в”ңв”Җв”Җ keyring_core.dart
    в””в”Җв”Җ src/
        в”ңв”Җв”Җ entry.dart           вҶҗ KeyringEntry class (matches Rust Entry)
        в”ңв”Җв”Җ exceptions.dart       вҶҗ KeyringException, KeyringErrorType enum
        в””в”Җв”Җ store.dart            вҶҗ KeyringStore abstract, CredentialPersistence enum
```

**Key Dart Types:**

| Rust `keyring-core` | Dart `keyring_core` |
|---------------------|---------------------|
| `Entry::new(s, u)` | `KeyringEntry(s, u)` |
| `Entry::set_password(p)` | `entry.setPassword(p)` |
| `Entry::get_password()` | `entry.getPassword()` |
| `Entry::delete_credential()` | `entry.deleteCredential()` |
| `Error::NoEntry` | `KeyringException(KeyringErrorType.noEntry, ...)` |
| `CredentialStoreApi` trait | `KeyringStore` abstract class |
| `CredentialPersistence` enum | `CredentialPersistence` enum |

### 2. `packages/keyring_native` вҖ” Native Implementation (Dart + Rust FFI)

**Purpose:** Provide native credential storage via Rust keyring crates compiled with `native_toolchain_rust`.

**Files:**
```
keyring_native/
в”ңв”Җв”Җ pubspec.yaml                     вҶҗ Depends on keyring_core, native_toolchain_rust, hooks
в”ңв”Җв”Җ hook/
в”Ӯ   в””в”Җв”Җ build.dart                   вҶҗ RustBuilder build hook (calls cargo)
в”ңв”Җв”Җ lib/
в”Ӯ   в”ңв”Җв”Җ keyring_native.dart          вҶҗ Library export
в”Ӯ   в””в”Җв”Җ src/
в”Ӯ       в”ңв”Җв”Җ native_store.dart        вҶҗ NativeKeyringStore implements KeyringStore
в”Ӯ       в””в”Җв”Җ ffi_bindings.g.dart      вҶҗ Auto-generated by native_toolchain_rust
в””в”Җв”Җ native/                          вҶҗ Rust source
    в”ңв”Җв”Җ Cargo.toml
    в”ңв”Җв”Җ rust-toolchain.toml          вҶҗ Targets: linux, windows, macos, ios, android
    в””в”Җв”Җ src/
        в”ңв”Җв”Җ lib.rs                   вҶҗ extern "C" FFI functions
        в””в”Җв”Җ platform.rs              вҶҗ Conditional compilation per target OS
```

**FFI API (Rust side):**

```rust
// lib.rs вҖ” extern "C" functions
keyring_init()                    вҶ’ i32 (0 = ok, negative = error)
keyring_set_password(service, user, password) вҶ’ i32
keyring_get_password(service, user, &out) вҶ’ i32
keyring_delete_password(service, user) вҶ’ i32
keyring_free_string(ptr)         вҶ’ void
```

**Platform Selection (Rust `platform.rs`):**

```rust
#[cfg(target_os = "linux")]
вҶ’ linux_keyutils_keyring_store::Store
#[cfg(target_os = "windows")]
вҶ’ windows_native_keyring_store::Store
#[cfg(target_os = "macos")]
вҶ’ keyring_core::mock::Store (placeholder for macOS Keychain)
```

**Build Configuration:**
- Targets: Linux (x86_64, aarch64), Windows (x86_64, aarch64), macOS (x86_64, aarch64), iOS (device + sim), Android (armv7, aarch64, x86_64)
- Rust edition: 2024
- Crate type: `["staticlib", "cdylib"]`

### 3. `packages/keyring_web` вҖ” Web Implementation (Pure Dart)

**Purpose:** Enable keyring on web platforms where native code cannot run. Uses IndexedDB for storage and Web Crypto API for encryption.

**Files:**
```
keyring_web/
в”ңв”Җв”Җ pubspec.yaml
в””в”Җв”Җ lib/
    в”ңв”Җв”Җ keyring_web.dart
    в””в”Җв”Җ src/
        в””в”Җв”Җ web_store.dart               вҶҗ WebKeyringStore implements KeyringStore
```

**Encryption:**
- AES-256-GCM using `dart:html` Web Crypto API
- Master key derived from platform-specific seed
- Credentials base64-encoded, encrypted, stored in IndexedDB

---

### 4. `packages/keyring` вҖ” Umbrella Package

**Purpose:** Single package users import (`dart pub add keyring`). Auto-selects the platform implementation.

**Files:**
```
keyring/
в”ңв”Җв”Җ pubspec.yaml
в””в”Җв”Җ lib/
    в”ңв”Җв”Җ keyring.dart                    вҶҗ Re-exports keyring_core, convenience API
    в””в”Җв”Җ src/
        в””в”Җв”Җ selector.dart               вҶҗ Platform detection and store selection
```

**Store Selection Logic:**

```dart
KeyringStore selectStore() {
  // Try native first (desktop/mobile)
  if (isNativePlatform) return NativeKeyringStore();
  // Fall back to web
  return WebKeyringStore();
}
```

## Build & Workflow

### Development Setup

```bash
# Ensure Rust toolchain is installed
rustup toolchain install 1.88.0

# Install required targets (example)
rustup target add x86_64-unknown-linux-gnu aarch64-unknown-linux-gnu
rustup target add x86_64-pc-windows-msvc aarch64-pc-windows-msvc
rustup target add aarch64-apple-darwin x86_64-apple-darwin

# Install workspace dependencies
dart pub get

# Build native Rust libraries (triggers native_toolchain_rust hook)
dart run build

# Run tests
dart test
```

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| FFI strategy | `native_toolchain_rust` | Auto-builds Rust, generates Dart bindings, no manual FFI |
| Web approach | Pure Dart (IndexedDB + Web Crypto) | No WASM needed, full browser compatibility |
| Android/iOS | Rust FFI via `native_toolchain_rust` | No Flutter plugins needed, Rust compiles for Android NDK + iOS |
| macOS Keychain | Future: `security-framework` Rust crate | Not yet cloned; mock placeholder for now |
| Error model | Matches Rust `keyring-core` exactly | Ensures API consistency across all implementations |
| Persistence mapping | Per platform | Linux: UntilReboot, Windows: UntilDelete, Web: UntilDelete |

## Questions for Later

2. **Android Keystore:** Do we write a Rust wrapper (`android-keystore-rs`) or use platform channels?
3. **iOS Keychain:** Similar question вҖ” Rust `security-framework` or Flutter plugin?
4. **Encrypted file fallback:** Should we provide a cross-platform encrypted file store for when native options are unavailable?
5. **API ergonomics:** Sync vs async methods for `keyring_native`? (FFI calls are sync, but entry operations should be async for consistency)
